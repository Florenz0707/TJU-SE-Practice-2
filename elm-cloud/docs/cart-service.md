# cart-service 微服务详情

## 1. 功能概述
`cart-service` (端口: 8086) 负责处理用户的购物车业务逻辑。其主要职责为：
- 维护当前用户在各商家的点餐购物车条目。
- 提供添加、修改（更新数量）以及删除购物车项的功能。

## 2. 数据库与表结构
- 数据库连接：使用独立或共享的订单类数据库 (`elm_order` 或 `elm_cart`)。
- 核心实体与表：
  - `Cart` 表包含：`id`, `user_id` (用户), `business_id` (商家), `food_id` (商品/食品), 和 `quantity` (数量)。
  - 注意：在微服务化过程中，将原本复杂的多对多/一对多级联关系彻底打平（Flatten），使之成为单纯存放各个聚合ID的记录，避免庞大查询带来的性能和耦合问题。

## 3. 提供的接口

### 3.1 外部暴露接口 (外部通过 Gateway 调用)
调用以 `/elm/api/carts` 路径为主：
- **GET /api/carts**：使用 `Authorization` 获取当前用户所有的购物车项。
- **POST /api/carts**：添加新商品到购物车，如果已存在则累加 `quantity`。
- **PATCH /api/carts/{id}**：更新某个购物车条目的数量。
- **DELETE /api/carts/{id}**：将指定 ID 的条目从购物车移除。

### 3.2 内部接口
为保持独立性与轻量化，暂不需要提供内部 Feign 调用，主要为前端服务。

## 4. 迁移细节与遇到的问题
- **实体关系打平的需求**：原有的老版实现可能带有极其深度的 JPA 联表级联查询。为了契合独立微服务架构并支持未来前端只请求其所需数据的需求，将购物车表从带有关联对象的结构变更为仅含有 `food_id` 和 `business_id` 基础属性的模型，并在数据库中移除了旧表的关联约束。
- **JWT 解析工具与 PowerShell 编码踩坑**：为了保持服务的轻量，复用了 JWT 提取器。在跨服务复制文件时，由于 PowerShell 的输出重定向引入了 UTF-8 (BOM) 头，导致 Maven 编译报 `非法字符: \ufeff`，后续已通过脚本统一去除 BOM 解决。
